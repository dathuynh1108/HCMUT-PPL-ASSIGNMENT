020/**
021 * Instances of this class are thrown by BCEL's class file verifier "JustIce" when
022 * a class file to verify does not pass the verification pass 3 because of a violation
023 * of a constraint that is usually only verified at run-time (pass 4).
024 * The Java Virtual Machine Specification, 2nd edition, states that certain constraints
025 * are usually verified at run-time for performance reasons (the verification of those
026 * constraints requires loading in and recursively verifying referenced classes) that
027 * conceptually belong to pass 3; to be precise, that conceptually belong to the
028 * data flow analysis of pass 3 (called pass 3b in JustIce).
029 * These are the checks necessary for resolution: Compare pages 142-143 ("4.9.1 The
030 * Verification Process") and pages 50-51 ("2.17.3 Linking: Verification, Preparation,
031 * and Resolution") of the above mentioned book.
032 * <B>TODO: At this time, this class is not used in JustIce.</B>
033 *
034 */
035public class LinkingConstraintException extends StructuralCodeConstraintException{
036
037    private static final long serialVersionUID = -5239226345026321126L;
038}
